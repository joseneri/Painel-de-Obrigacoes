# Template de Plano de Pesquisa

Use antes de implementar tarefa media/complexa, ou quando houver tecnologia,
API, arquitetura, comportamento recente ou decisao que possa estar
desatualizada.

## Pergunta Principal

Qual decisao ou problema esta pesquisa precisa resolver?

## Perguntas de Apoio

- O que a documentacao oficial recomenda?
- Quais trade-offs existem?
- Quais edge cases sao conhecidos?
- Quais alternativas foram descartadas?
- O que se aplica ao contexto deste projeto?

## Criterios de Avaliacao

- Compatibilidade com `docs/architecture.md`.
- Simplicidade e manutencao.
- Performance e seguranca.
- Testabilidade.
- Custo de implementacao.
- Risco de lock-in ou complexidade acidental.

## Plano de Execucao da Pesquisa

1. Buscar fontes oficiais.
2. Comparar alternativas.
3. Procurar edge cases e problemas recorrentes.
4. Registrar achados em `tmp/achados-*.md`.
5. Gerar recomendacao.
6. Criar plano de desenvolvimento baseado nos achados.
